= Solution NetApp avec charges de travail de la plateforme de conteneurs Red Hat OpenShift dans le cloud hybride
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Les clients peuvent être à un stade de leur parcours de modernisation où ils sont prêts à déplacer certaines charges de travail sélectionnées ou toutes les charges de travail de leurs centres de données vers le cloud.  Ils peuvent choisir d’utiliser des conteneurs OpenShift autogérés et un stockage NetApp autogéré dans le cloud pour diverses raisons.  Ils doivent planifier et déployer la plateforme de conteneurs Red Hat OpenShift (OCP) dans le cloud pour un environnement prêt pour la production afin de migrer leurs charges de travail de conteneurs depuis leurs centres de données.  Leurs clusters OCP peuvent être déployés sur VMware ou Bare Metal dans leurs centres de données et sur AWS, Azure ou Google Cloud dans l'environnement cloud.

NetApp Cloud Volumes ONTAP offre protection des données, fiabilité et flexibilité pour les déploiements de conteneurs dans AWS, Azure et Google Cloud. Trident sert de provisionneur de stockage dynamique pour consommer le stockage persistant Cloud Volumes ONTAP pour les applications avec état des clients. Trident Protect peut être utilisé pour les exigences de gestion des données des applications avec état, telles que la protection des données, la migration et la continuité d'activité.
